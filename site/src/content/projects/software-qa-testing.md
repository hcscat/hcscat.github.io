## 프로젝트 개요

개발 전환 이전에는 소프트웨어 테스터와 QA/QC 역할을 수행했습니다. 솔루션, 웹, 모바일 앱 환경에서 요구사항, 화면 흐름, 데이터 상태, 서버 환경을 함께 확인하며 품질 검증 업무를 진행했습니다.

## 수행한 일

- 요구사항과 화면 흐름을 기준으로 테스트케이스를 작성하고 수행했습니다.
- 발견한 결함은 재현 절차, 기대 결과, 실제 결과, 발생 조건을 정리해 추적했습니다.
- Linux 서버와 DB 상태를 확인하며 화면 오류와 데이터 오류의 원인 범위를 좁혔습니다.
- 릴리스 전 회귀 테스트를 통해 기존 기능 영향 여부를 확인했습니다.

## 개발 경력과 연결되는 점

QA/QC 경험은 현재 개발 업무에서 변경 영향도 확인, 예외 케이스 검토, 테스트 산출물 작성, 운영 이슈 분석 관점으로 이어지고 있습니다. 기능 구현뿐 아니라 운영 중 발생할 수 있는 조건을 함께 확인하는 태도의 기반입니다.
