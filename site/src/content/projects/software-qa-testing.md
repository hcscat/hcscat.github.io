## 문제 환경

QA 업무에서는 기능이 동작하는지 확인하는 것뿐 아니라, 어떤 조건에서 문제가 재현되는지 정확히 정리하는 일이 중요했습니다. 솔루션, 웹, 모바일 앱 환경을 검증하면서 요구사항, 화면 흐름, 데이터 상태, 서버 환경을 함께 확인했습니다.

## 수행한 일

- 요구사항과 화면 흐름을 기준으로 테스트케이스를 작성하고 수행했습니다.
- 발견한 결함은 재현 절차, 기대 결과, 실제 결과, 발생 조건을 정리해 추적했습니다.
- Linux 서버와 DB 상태를 확인하며 화면 오류와 데이터 오류의 원인 범위를 좁혔습니다.
- 릴리스 전 회귀 테스트를 통해 기존 기능 영향 여부를 확인했습니다.

## 개발 경력과 연결되는 점

이 경험은 현재 개발 업무에서 변경 영향도 확인, 예외 케이스 검토, 단위테스트 산출물 작성, 운영 이슈 분석 관점으로 이어집니다. 포트폴리오에서는 특정 제품이나 고객 정보를 공개하지 않고, 개발자로 전환한 뒤에도 유지되는 품질 관점의 강점으로 정리합니다.
