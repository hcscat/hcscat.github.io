채용공고 검색은 기술명을 많이 넣는다고 좋아지지 않습니다. 내 경력과 맞는 직무, 도메인, 운영 방식, 사용 기술을 함께 넣어야 실제로 검토할 만한 공고가 줄어듭니다.

## 우선 검색 축

- Java Spring 업무 시스템 개발
- Java Spring 운영개발
- Java MyBatis 공공 시스템
- Java JSP Nexacro
- exBuilder Java Oracle
- SAP UI5 Fiori JavaScript
- 이커머스 운영개발
- 백오피스/관리자 화면 개발

## 제외 또는 감점 축

- React/TypeScript 단독 프론트엔드
- Node.js 단독 백엔드
- DevOps/SRE 중심
- 모바일 앱 전담
- AI 리서치/데이터 사이언스 전담

## 구조화한 이유

채용사이트마다 필터와 검색 방식이 다르기 때문에 수작업으로 보면 누락과 중복이 생깁니다. 그래서 검색어, 제외 조건, 우선 검토 조건을 문서화해 두고 공고를 볼 때 같은 기준으로 비교하려고 합니다. 이 글은 구현 결과물이 아니라 공개 가능한 수준의 검토 기준 메모입니다.
