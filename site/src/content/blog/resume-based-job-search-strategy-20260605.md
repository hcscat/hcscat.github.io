채용공고 검색은 기술명을 많이 넣는다고 좋아지지 않습니다. 내 경력과 맞는 직무, 도메인, 운영 방식, 사용 기술을 함께 넣어야 실제로 검토할 만한 공고가 줄어듭니다.

## 우선 검색 축

- Java Spring 업무 시스템 개발
- Java Spring 운영개발
- Java MyBatis 공공 시스템
- Java JSP Nexacro
- exBuilder Java Oracle
- SAP UI5 Fiori JavaScript
- 이커머스 운영개발
- 백오피스/관리자 화면 개발

## 제외 또는 감점 축

- React/TypeScript 단독 프론트엔드
- Node.js 단독 백엔드
- DevOps/SRE 중심
- 모바일 앱 전담
- AI 리서치/데이터 사이언스 전담

## 시스템화한 이유

채용사이트마다 필터와 검색 방식이 다르기 때문에 수작업으로 보면 누락과 중복이 생깁니다. 그래서 이력서 기반 프로필, 검색어 생성, 사이트별 수집, 정규화, 적합도 점수, 검토 UI로 나누어 개인 채용 인텔리전스 플랫폼을 만들고 있습니다.
